## 4. BlockAllocator 接口定义

```cpp
// slots/block_allocator.h
#ifndef CABE_BLOCK_ALLOCATOR_H
#define CABE_BLOCK_ALLOCATOR_H

#include "common/structs.h"

#include <concepts>
#include <cstddef>
#include <cstdint>
#include <span>

namespace cabe {

    template<typename T>
    concept BlockAllocator = requires(T& alloc, const T& calloc,
                                       DeviceId dev, std::uint64_t block_count,
                                       BlockId id, BlockId* out,
                                       std::span<const BlockId> active_blocks) {
        { alloc.Init(dev, block_count) }                             -> std::same_as<int32_t>;
        { alloc.Acquire(out) }                                       -> std::same_as<int32_t>;
        { alloc.Recycle(id) }                                        -> std::same_as<void>;
        { calloc.Available() }                                       -> std::convertible_to<std::size_t>;
        { calloc.Empty() }                                           -> std::same_as<bool>;
        { alloc.RebuildFromActive(dev, block_count, active_blocks) } -> std::same_as<int32_t>;
    };

} // namespace cabe

#endif // CABE_BLOCK_ALLOCATOR_H
```

**设计要点**：
- 全部返回 `int32_t` 错误码（内部组件约定）；`Recycle` 返回 `void`（回收不失败——队列容量 = 总块数 + 1，不可能满溢）。
- `Available` 用 `std::convertible_to`（允许 `size_t` 或 `uint64_t`）。
- `RebuildFromActive` 接收 `std::span<const BlockId>`——P5 恢复时从 MetaIndex 提取已用块列表传入。
- 纯数据结构——不涉及 I/O，不依赖 IoBackend。TRIM 由 Engine 层负责。

## 5. RingBlockAllocator 设计

### 5.1 类声明

```cpp
// slots/ring/ring_block_allocator.h
#ifndef CABE_RING_BLOCK_ALLOCATOR_H
#define CABE_RING_BLOCK_ALLOCATOR_H

#include "slots/block_allocator.h"
#include "common/error_code.h"
#include "common/structs.h"

#include <cstddef>
#include <cstdint>
#include <span>

namespace cabe {

    class RingBlockAllocator {
    public:
        RingBlockAllocator() noexcept = default;
        ~RingBlockAllocator();

        RingBlockAllocator(const RingBlockAllocator&) = delete;
        RingBlockAllocator& operator=(const RingBlockAllocator&) = delete;
        RingBlockAllocator(RingBlockAllocator&& other) noexcept;
        RingBlockAllocator& operator=(RingBlockAllocator&& other) noexcept;

        int32_t Init(DeviceId dev, std::uint64_t block_count);
        int32_t Acquire(BlockId* out);
        void Recycle(BlockId id);
        std::size_t Available() const noexcept;
        bool Empty() const noexcept;
        int32_t RebuildFromActive(DeviceId dev, std::uint64_t block_count,
                                   std::span<const BlockId> active_blocks);

    private:
        // 注：当前 Init 从块 0 开始分配。P5 引入超级块后，设备头部将独立
        // 预留为元数据区域（非数据块），届时需调整起始块号或偏移计算。
        // 详见 doc/P4/P4M3_tsan_deploy_design.md §6 和 doc/P4.5/README.md §关键技术备忘 第 7 条。
        BlockId* slots_ = nullptr;
        std::size_t capacity_ = 0;
        std::size_t head_ = 0;
        std::size_t tail_ = 0;
    };

    static_assert(BlockAllocator<RingBlockAllocator>);

} // namespace cabe

#endif // CABE_RING_BLOCK_ALLOCATOR_H
```

### 5.2 实现要点

**Init**（首次打开，全部块空闲）：
```cpp
int32_t RingBlockAllocator::Init(DeviceId dev, std::uint64_t block_count) {
    delete[] slots_;
    capacity_ = block_count + 1;
    slots_ = new BlockId[capacity_];
    head_ = 0;
    tail_ = 0;

    for (std::uint64_t i = 0; i < block_count; ++i) {
        slots_[tail_] = BlockId::Make(dev, i);
        tail_ = (tail_ + 1) % capacity_;
    }
    return err::kSuccess;
}
```

容量 = block_count + 1：浪费一个槽位区分满/空（D4 锁定）。从块 0 到 block_count-1 依次入队。

**Acquire**（获取一个可用块号）：
```cpp
int32_t RingBlockAllocator::Acquire(BlockId* out) {
    if (head_ == tail_) return err::kEngineNoSpace;
    *out = slots_[head_];
    head_ = (head_ + 1) % capacity_;
    return err::kSuccess;
}
```

从队列头部取出，O(1)。空队列（head == tail）返回 `kEngineNoSpace`。

**Recycle**（回收一个块号）：
```cpp
void RingBlockAllocator::Recycle(BlockId id) {
    slots_[tail_] = id;
    tail_ = (tail_ + 1) % capacity_;
}
```

放到队列尾部，O(1)。FIFO 语义——刚回收的块排到最后，其他块都用过一轮才轮到它。

**Available / Empty**：
```cpp
std::size_t RingBlockAllocator::Available() const noexcept {
    return (tail_ - head_ + capacity_) % capacity_;
}

bool RingBlockAllocator::Empty() const noexcept {
    return head_ == tail_;
}
```

**RebuildFromActive**（P5 恢复用——从已用块列表反推空闲块）：
```cpp
int32_t RingBlockAllocator::RebuildFromActive(DeviceId dev, std::uint64_t block_count,
                                                std::span<const BlockId> active_blocks) {
    delete[] slots_;
    capacity_ = block_count + 1;
    slots_ = new BlockId[capacity_];
    head_ = 0;
    tail_ = 0;

    // 临时位图：标记已用块
    std::vector<bool> used(block_count, false);
    for (const auto& bid : active_blocks) {
        used[bid.block_idx()] = true;
    }

    // 未标记的块入队
    for (std::uint64_t i = 0; i < block_count; ++i) {
        if (!used[i]) {
            slots_[tail_] = BlockId::Make(dev, i);
            tail_ = (tail_ + 1) % capacity_;
        }
    }
    return err::kSuccess;
}
```

临时位图在方法返回时自动销毁（`std::vector<bool>` 的生命周期限于方法内）。

**析构**：
```cpp
RingBlockAllocator::~RingBlockAllocator() {
    delete[] slots_;
}
```

**Move 语义**：
```cpp
RingBlockAllocator::RingBlockAllocator(RingBlockAllocator&& other) noexcept
    : slots_(other.slots_)
    , capacity_(other.capacity_)
    , head_(other.head_)
    , tail_(other.tail_) {
    other.slots_ = nullptr;
    other.capacity_ = 0;
    other.head_ = 0;
    other.tail_ = 0;
}
```

转移指针，旧对象置空。Move 赋值同理（先释放自身资源）。

## 7. 契约测试设计

```cpp
template<typename T>
class BlockAllocatorContractTest : public ::testing::Test {
protected:
    T allocator_;
};

using BlockAllocatorImpls = ::testing::Types<cabe::RingBlockAllocator>;
TYPED_TEST_SUITE(BlockAllocatorContractTest, BlockAllocatorImpls);
```

**用例清单**（全部用 `TYPED_TEST`——未来自定义分配器加入类型列表即可复用）：

| 用例 | 验证 |
|---|---|
| `InitFillsAll` | Init 后 Available() == block_count，Empty() == false |
| `AcquireReturnsBlock` | Acquire 成功返回 kSuccess + 有效 BlockId |
| `AcquireExhausted` | 全部取出后 Acquire 返回 kEngineNoSpace |
| `RecycleRestores` | 回收后 Available 增加，可再次获取 |
| `FifoOrder` | 获取顺序与 Init 填入顺序一致——验证 FIFO 语义 |
| `RecycledGoesToTail` | 回收的块排到队尾——不是立即被下一次 Acquire 取到 |
| `DeviceIdPreserved` | Init 传入的 device_id 在 Acquire 返回的 BlockId 中保留 |
| `RebuildFromActive` | 传入已用块列表后 Available == block_count - active_count |
| `RebuildExcludesActive` | 重建后 Acquire 返回的块不包含已用块 |
| `MoveConstruct` | move 后新对象可用（Available 正确），旧对象为空 |
| `ConceptSatisfied` | `static_assert(BlockAllocator<RingBlockAllocator>)` 编译通过 |

## 8. 风险与权衡

| 风险 | 说明 | 缓解 |
|---|---|---|
| `new BlockId[]` 动态分配 | Init 和 RebuildFromActive 各一次堆分配 | 只在打开设备 / 恢复时发生，不在热路径；P7 无锁阶段不涉及分配 |
| `std::vector<bool>` 位图 | RebuildFromActive 的临时位图用标准库实现，可能有位压缩的性能开销 | 恢复不在热路径；128GB 设备位图仅 16KB；如需极致性能可改自实现位数组 |
| Init 从块 0 开始 | P5 超级块需要调整 | 代码和文档已注释说明改造方向 |
| 与旧 FreeList 并行存在 | P4.5M1 不删旧代码——两套分配器并存 | P4.5M2 Engine 切换后删旧的 |
