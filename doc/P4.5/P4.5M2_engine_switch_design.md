# Cabe P4.5-M2 设计：Engine 切换 + 旧代码清理

> 本里程碑将 Engine 从直接调用朴素 FreeList 切换到通过 P4.5M1 定义的 BlockAllocator
> 抽象层调用，并让 CMake `CABE_BLOCK_ALLOCATOR` 变量真正控制编译期分派。切换后功能
> 等价于 P4，公开 API 不变。与 P3M3（IoBackend / MetaIndex 切换）完全对称。
>
> **本文为详细设计**；其中 C++ 片段为设计示意，代码实装阶段以此为准。

## 1. 目标与范围

### 1.1 目标

1. `engine/backend_config.h` 加块分配器分派（`#if CABE_USE_BLOCK_RING_QUEUE`）。
2. DeviceContext 改造：`FreeList free_list` → `BlockAllocatorImpl block_allocator`。
3. Engine 改造：`free_list.Allocate` → `block_allocator.Acquire`，`free_list.Free` → `block_allocator.Recycle`。
4. TRIM 占位：Engine 内部加 `TrimDeviceBlock` 空函数，Delete 路径调用。
5. `engine/CMakeLists.txt` 加 `CABE_BLOCK_ALLOCATOR` 编译期分派。
6. 删除旧代码：`engine/free_list.h` / `engine/free_list.cpp` / `test/engine/free_list_test.cpp`。

### 1.2 交付范围

1. **`engine/backend_config.h`**（修改）：加块分配器分派分支。
2. **`engine/device_context.h`**（修改）：持有 `BlockAllocatorImpl`。
3. **`engine/engine.h`**（修改）：加 `TrimDeviceBlock` 私有方法声明。
4. **`engine/engine.cpp`**（修改）：切换到抽象层调用 + TRIM 占位。
5. **`engine/CMakeLists.txt`**（修改）：移除 `free_list.cpp`、链接 `cabe_slots`、加编译定义。
6. **`test/CMakeLists.txt`**（修改）：移除 `test_free_list` 目标。
7. **删除**：`engine/free_list.h`、`engine/free_list.cpp`、`test/engine/free_list_test.cpp`。

### 1.3 推迟范围

| 推迟项 | 落点 | 原因 |
|---|---|---|
| 异步批量 TRIM | **P7** | 当前只占位；P7 利用 reactor 事件循环异步批量发送 |
| 超级块 | **P5** | Init 从块 0 开始；P5 改为设备头部独立预留 |

## 2. 决策汇总

| 编号 | 决策 | 结果 | 理由 |
|---|---|---|---|
| **P4.5M2-D1** | TRIM 占位形式 | 空函数 `TrimDeviceBlock`，内部 TODO 注释 | 比纯注释更明确（调用点可见）；P7 实现时填充函数体即可 |

## 3. backend_config.h 改造

在现有 IoBackend / MetaIndex 分派之后，追加块分配器分派：

```diff
+ // ---- BlockAllocator 分派 ----
+ #if defined(CABE_USE_BLOCK_RING_QUEUE)
+ #include "slots/ring/ring_block_allocator.h"
+ namespace cabe { using BlockAllocatorImpl = RingBlockAllocator; }
+ #else
+ #error "未选择 BlockAllocator：请设置 -DCABE_BLOCK_ALLOCATOR=ring_queue"
+ #endif

  #include "io/io_backend.h"
  #include "index/meta_index.h"
+ #include "slots/block_allocator.h"
  static_assert(cabe::IoBackend<cabe::IoBackendImpl>);
  static_assert(cabe::MetaIndexBackend<cabe::MetaIndexImpl>);
+ static_assert(cabe::BlockAllocator<cabe::BlockAllocatorImpl>);
```

## 4. DeviceContext 改造

### 4.1 改造后

```cpp
// engine/device_context.h
#ifndef CABE_DEVICE_CONTEXT_H
#define CABE_DEVICE_CONTEXT_H

#include "engine/backend_config.h"
#include "engine/buffer_pool.h"

namespace cabe {

    struct DeviceContext {
        IoBackendImpl io;
        BufferPool pool{0};
        BlockAllocatorImpl block_allocator;
        MetaIndexImpl meta_index;
    };

} // namespace cabe

#endif // CABE_DEVICE_CONTEXT_H
```

### 4.2 变更对照

| 字段 | 改造前 | 改造后 |
|---|---|---|
| 块分配器 | `FreeList free_list`（`engine/free_list.h`） | `BlockAllocatorImpl block_allocator`（`slots/ring/ring_block_allocator.h`） |
| 其他 | 不变 | 不变 |

### 4.3 影响

- `#include "engine/free_list.h"` 移除——通过 `backend_config.h` 引入 `RingBlockAllocator`。
- `RingBlockAllocator` 支持 move 语义——`std::vector<DeviceContext>` 的 `push_back` 正常工作。

## 5. Engine 改造

### 5.1 Engine::Open

```diff
- dc.free_list.Init(0, dc.io.BlockCount());
+ dc.block_allocator.Init(0, dc.io.BlockCount());
```

### 5.2 Engine::Put

```diff
  // 覆盖写：key 已存在则先释放旧块
- if (dc.meta_index.Lookup(key, &old_meta) == err::kSuccess) {
-     dc.free_list.Free(old_meta.block);
- }
+ if (dc.meta_index.Lookup(key, &old_meta) == err::kSuccess) {
+     dc.block_allocator.Recycle(old_meta.block);
+ }

  // 分配新块
- int32_t rc = dc.free_list.Allocate(&block_id);
+ int32_t rc = dc.block_allocator.Acquire(&block_id);

  // 写失败回收
- dc.free_list.Free(block_id);
+ dc.block_allocator.Recycle(block_id);
```

### 5.3 Engine::Delete

```diff

+ dc.block_allocator.Recycle(meta.block);
+ TrimDeviceBlock(dc, meta.block);
  dc.meta_index.Delete(key);
```
