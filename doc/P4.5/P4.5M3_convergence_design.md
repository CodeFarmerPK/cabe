## 2. 收敛技术索引（薄索引形态）

### 2.1 块分配器抽象层（详 [P4.5M1_block_allocator_design.md](P4.5M1_block_allocator_design.md)）

- **BlockAllocator C++20 接口**：6 个方法——`Init` / `Acquire` / `Recycle` / `Available` / `Empty` / `RebuildFromActive`。纯数据结构，不涉及 I/O。
- **RingBlockAllocator**：固定大小环形队列（FIFO），容量 = block_count + 1（浪费一个槽位区分满/空）。
- **`Acquire` / `Recycle` 命名**：取代 Allocate / Free，避免与内存操作混淆。
- **`RebuildFromActive`**：临时位图实现——P5 恢复时从已用块列表反推空闲块，方法返回后位图销毁。
- **目录结构**：`slots/block_allocator.h`（接口）+ `slots/ring/ring_block_allocator.*`（实现）。
- **CMake**：`CABE_BLOCK_ALLOCATOR=ring_queue` 编译期切换。
- **契约测试**：`TYPED_TEST` 形态，11 个用例，与 MetaIndex 对称。
- **决策锁定**：P4.5M1-D1（Init 从块 0）、D2（TYPED_TEST）、D3（临时位图）。

### 2.2 Engine 切换 + 旧代码清理（详 [P4.5M2_engine_switch_design.md](P4.5M2_engine_switch_design.md)）

- **DeviceContext 改造**：`FreeList free_list` → `BlockAllocatorImpl block_allocator`。
- **Engine 改造**：`free_list.Allocate` → `block_allocator.Acquire`，`free_list.Free` → `block_allocator.Recycle`。
- **TRIM 占位**：`TrimDeviceBlock` 空函数，Delete 路径调用，P7 填充异步批量逻辑。
- **配置头**：`engine/backend_config.h` 加块分配器分派（`CABE_USE_BLOCK_RING_QUEUE`）+ `static_assert` 验证。
- **旧代码删除**：`engine/free_list.h` / `engine/free_list.cpp` / `test/engine/free_list_test.cpp`。
- **决策锁定**：P4.5M2-D1（TRIM 占位用空函数 `TrimDeviceBlock`）。

### 2.4 关键技术备忘

1. **升序分配无性能价值**：1M 块在 NVMe SSD 上升序不连续分配与随机分配无可观测差异。
2. **空闲块号自然趋向无序**：回收顺序取决于用户删除行为，不可控。
3. **FIFO 对 SSD 磨损更友好**：块使用均匀分散。
4. **环形队列无锁扩展路径**：P7 阶段 `head_` / `tail_` 改为原子变量。
5. **块分配器不持久化**：恢复时从 MetaIndex 反推，毫秒级纯内存操作。
6. **超级块**（P5）：届时设备头部独立预留为元数据区域，`Init` 起始块号需调整。
